get /positions

Lista, orderna y pagina las posiciones segun las opciones especificadas.

  • Request
  • Response

Headers

  • X-Session-Token: required (string)

    Token de acceso

    Example:

    q16d7NHS2b3M8OTgTKqbgBoRKsZahUWqk3tuTujHqjETS6dq
  • X-OS: required (string)

    Sistema operativo

    Example:

    android
  • X-imei: required (string)

    IMEI de dispositivo

    Example:

    ZXKH34V

Query Parameters

  • page: (integer)

    Pagina actual

    Example:

    1
  • limit: (integer)

    Numero de resultados por pagina

    Example:

    10
  • filters: (string)

    Json indicando los filtros

    Example:

    [
  {
    "field": "updatetime",
    "comparison": "gte",
    "value": "2015-07-07"
  },
  {
    "field": "updatetime",
    "comparison": "lte",
    "value": "2015-07-08"
  }
]
  • sort: (string)

    Json indicando los ordenamientos

    Example:

    [
  {
    "field": "updatetime",
    "direction": "ASC"
  }
]

HTTP status code 200

    "total": 10,

    ]
  }              
}

post /positions

Almacena un posicion con la informacion reportada en el json de entrada.

  • Request
  • Response

Headers

  • X-Session-Token: required (string)

    Token de acceso

    Example:

    q16d7NHS2b3M8OTgTKqbgBoRKsZahUWqk3tuTujHqjETS6dq
  • X-OS: required (string)

    Sistema operativo

    Example:

    android
  • X-imei: required (string)

    IMEI de dispositivo

    Example:

    ZXKH34V

Query Parameters

  • values: required (string)

    Json con la posicion que se debe almacenar

    Example:

    {
  "latitude": 4.712,
  "longitude": -74.454,
  "updatetime": "2015-07-07 16:35:00"        
}

HTTP status code 200

Body

Type: application/json

Example:

{
  "error": false,
  "msg": "OK",
  "data": {
    "id_position": 11
  }
}

post /login

Verifica el usuario y contraseña dados. Si son correctos genera un access token y un refresh token.

  • Request
  • Response

Headers

  • X-OS: required (string)

    Sistema operativo

    Example:

    android
  • X-imei: required (string)

    IMEI de dispositivo

    Example:

    ZXKH34V

Query Parameters

  • values: required (string)

    Json con informacion de login y password

    Example:

    {
  "user": "juan",
  "password": "e10adc3949ba59abbe56e057f20f883e"
}

HTTP status code 200

El usuario y contraseña son correctos. Se entrega access token y refresh token.

Body

Type: application/json

Example:

{
  "error": false,
  "msg": "OK",
  "access_token": "uydBWqWX0d55a0SOnp1v8uK0nRKEAChiJl2gh8fpdBD5YOp7",
  "refresh_token": "ynhRpou9wrJllCfhQvKiverfim3sF0ONCJqAzF7FxnBpYZ5E",
  "valid_until": null              
}
